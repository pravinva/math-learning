Topic 1 · Week 3
Arrays, Vectors & Matrices
Storing many values under one name — the core MATLAB data structure
MATLAB is built for arrays. A vector is a list of numbers; a matrix is a 2-D grid. Instead of 100 separate variables you keep one array and operate on it all at once. The single most important rule: MATLAB indexing starts at 1, not 0.
DefinitionCreating vectors and matrices▶
r = [1 2 3 4]; % row vector c = [1; 2; 3]; % column vector (semicolons = new row) M = [1 2 3; 4 5 6]; % 2x3 matrix z = zeros(3); % 3x3 of zeros o = ones(2,4); % 2x4 of ones
PatternThe colon operator & ranges▶

The colon : builds evenly-spaced sequences — used constantly for loops and slicing.

1:5 % [1 2 3 4 5] 0:2:10 % [0 2 4 6 8 10] (start:step:end) 10:-1:1 % counts down linspace(0,1,5) % 5 evenly spaced points 0 to 1
PatternIndexing — starts at 1▶
v = [10 20 30 40 50]; v(1) % 10 (FIRST element is index 1) v(end) % 50 (last element) v(2:4) % [20 30 40] (a slice) M(2,3) % row 2, column 3 M(:,1) % entire first column
The biggest beginner trap: MATLAB is 1-indexed. v(0) is an error. (Python, in Guide 3, is 0-indexed — don't mix them up.)
ExampleWorked example — slicing a matrix▶
Example — Extract a value and a row

For M = [1 2 3; 4 5 6; 7 8 9], find M(3,2) and M(1,:).

1
M(3,2) = row 3, col 2 = 8.
2
M(1,:) = all of row 1 = [1 2 3].
ANSWER 8  and  [1 2 3]

Topic 2 · Week 3
Cell & 3-D Arrays
When a plain numeric grid is not enough
A normal array must hold one type — all numbers, or all text. Sometimes you need to mix: a name, an age and a score together. That is what a cell array (curly braces {}) is for. And when data is layered — like a stack of images — a 3-D array adds "pages" beyond rows and columns.
DefinitionCell arrays — mixed types▶

Cell arrays use curly braces and can hold any type in each cell.

c = {'Sid', 20, [90 85 88]}; c{1} % 'Sid' (curly braces get CONTENTS) c{3} % [90 85 88] c(1) % {'Sid'} (round braces get a sub-cell)
Braces matter: c{i} extracts the contents of cell i; c(i) returns another cell. Use curly braces to actually work with the value inside.
Definition3-D arrays — pages▶

A 3-D array is indexed by row, column, page. A colour image, for example, is height × width × 3 (red, green, blue pages).

A = zeros(2,2,3); % 2x2 with 3 pages A(:,:,1) = [1 2; 3 4]; % fill page 1 size(A) % [2 2 3]
ExampleWorked example — a record in a cell array▶
Example — Read a student record

Given rec = {'Ava', 19, true}, get the age and print a sentence.

age = rec{2}; % 19 fprintf('%s is %d\n', rec{1}, age);
OUTPUT Ava is 19

Topic 3 · Week 4
Flow Charts & Conditionals
Making a program choose between paths
Real algorithms make decisions: "if the temperature is too high, sound the alarm." A flow chart sketches that logic visually before you code it; the if statement turns it into MATLAB. Planning with a flow chart first is the habit that separates clean programs from spaghetti.
DefinitionFlow chart symbols▶
SymbolMeans
Oval (terminator)Start / End
ParallelogramInput / Output
RectangleProcess / calculation
DiamondDecision (yes/no)
ArrowFlow direction
Patternif / elseif / else▶
if mark >= 50 disp('Pass') elseif mark >= 40 disp('Supplementary') else disp('Fail') end

MATLAB checks each condition top-down and runs the first block that is true, then skips the rest. end closes the block.

Patternswitch — choosing among fixed cases▶
switch day case 'Sat' disp('Weekend') case 'Sun' disp('Weekend') otherwise disp('Weekday') end
When to use which: switch is cleaner than a long if/elseif chain when you are comparing one variable against several fixed values.
ExampleWorked example — classify a number▶
Example — Sign of a number

Write logic that prints whether x is positive, negative or zero, for x = -4.

if x > 0 disp('positive') elseif x < 0 disp('negative') else disp('zero') end
OUTPUT negative

Topic 4 · Week 4
Loops: for & while
Repeating work without copy-pasting
Computers excel at doing the same thing thousands of times. A for loop repeats a known number of times; a while loop repeats until a condition changes. Loops with an accumulator (a running total) are how you sum, count, search and average — the backbone of almost every algorithm.
DefinitionThe for loop▶
for i = 1:5 fprintf('%d squared is %d\n', i, i^2) end

The loop variable i takes each value in 1:5 in turn. Use a for loop when you know in advance how many iterations you need.

DefinitionThe while loop▶
n = 10; while n > 0 disp(n) n = n - 1; % MUST change n or loop never ends end
Infinite-loop trap: a while loop runs until its condition becomes false. If nothing inside changes the condition, it runs forever. Always update the controlling variable.
PatternAccumulator pattern & nested loops▶

Sum the numbers 1 to 100 with a running total:

total = 0; for k = 1:100 total = total + k; end % total = 5050

Nested loops (a loop inside a loop) sweep 2-D ranges — e.g. filling a matrix or a times-table:

for r = 1:3 for c = 1:3 T(r,c) = r * c; end end
ExampleWorked example — count down with while▶
Example — How many times does the loop run?

For n = 8; while n > 2, n = n - 2; end, how many iterations?

1
n=8 > 2 → run, n=6.
2
n=6 > 2 → run, n=4.
3
n=4 > 2 → run, n=2.
4
n=2 is not > 2 → stop.
ANSWER 3 iterations (final n = 2)

Topic 5 · Week 5
Functions & Modular Design
Packaging logic into reusable, testable units
As programs grow, putting everything in one long script becomes unmanageable. A function takes inputs, does one well-defined job, and returns outputs. Breaking a problem into functions — modular design — makes code easier to read, test and reuse. This is a core engineering skill.
DefinitionFunction syntax▶
circle_area.m
function A = circle_area(r) % Returns the area of a circle of radius r A = pi * r^2; end

Call it like any built-in: a = circle_area(5); returns 78.54. The output name (A) is whatever the function assigns before end.

Multiple outputs: function [s, p] = stats(a, b) returns two values; call as [sum, prod] = stats(3, 4);.
DefinitionScope — local variables▶

Variables created inside a function are local: they exist only while the function runs and cannot be seen by the rest of your program. The only way data gets in is through inputs, and out through outputs. This isolation is what makes functions safe to reuse.

PatternModular vs non-modular programming▶
Non-modularModular
One long scriptSmall focused functions
Hard to test in partsEach function tested alone
Copy-paste to reuseCall the function again
One bug breaks everythingBugs isolated to a unit
Rule: a good function does one thing and has a name that says what it does. If you can't name it simply, it is doing too much.
ExampleWorked example — a function with two outputs▶
Example — Min and max of a vector
function [lo, hi] = range_of(v) lo = min(v); hi = max(v); end % Call: [a, b] = range_of([4 9 1 7]); % a=1, b=9
ANSWER lo = 1, hi = 9

Topic 6 · Week 5
Recursion
A function that solves a problem by calling itself
Some problems are naturally defined in terms of smaller versions of themselves. Recursion is a function calling itself on a smaller input until it hits a base case that stops the chain. Factorial and the Fibonacci sequence are the classic examples — and favourite exam questions.
DefinitionThe two parts of recursion▶
1
Base case — the simplest input, answered directly, with no further call. This stops the recursion.
2
Recursive case — the function calls itself on a smaller input and builds the answer from the result.
Missing base case = infinite recursion → MATLAB errors with "maximum recursion limit reached". Always make sure each call moves closer to the base case.
ExampleFactorial: n! = n × (n−1)!▶
\[ n! = n \times (n-1)! \qquad\text{with}\qquad 0! = 1 \]
factorial_r.m
function f = factorial_r(n) if n <= 1 f = 1; % base case else f = n * factorial_r(n-1); % recursive case end end

Tracing factorial_r(4): 4 × (3 × (2 × (1))) = 4 × 3 × 2 × 1 = 24.

ExampleFibonacci: F(n) = F(n−1) + F(n−2)▶
\[ F(n) = F(n-1) + F(n-2) \qquad F(1)=1,\; F(2)=1 \]
function f = fib(n) if n <= 2 f = 1; % two base cases else f = fib(n-1) + fib(n-2); end end

The sequence is 1, 1, 2, 3, 5, 8, 13, … so fib(7) = 13.
