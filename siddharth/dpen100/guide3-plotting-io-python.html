Topic 1 · Week 6
2-D Plotting
Turning vectors into graphs
A table of numbers hides patterns; a graph reveals them instantly. The plot command is the engineer's most-used visualisation tool. The recipe is always the same: make an x-vector, make a matching y-vector, then plot and label. An unlabelled graph is worthless — always add a title and axis labels.
DefinitionThe basic plot recipe▶
x = 0:0.1:2*pi; % x-values y = sin(x); % matching y-values plot(x, y); xlabel('x (rad)'); ylabel('sin(x)'); title('Sine wave'); grid on;
Watch out: the x and y vectors must be the same length. Compute y from x (e.g. y = sin(x)) so they always match.
PatternMultiple lines with hold on▶
plot(x, sin(x), 'b-'); % blue solid hold on; % keep this figure, add more plot(x, cos(x), 'r--'); % red dashed legend('sin', 'cos'); hold off;

Without hold on, each plot replaces the previous one. Line specs combine colour + style: 'b-', 'r--', 'g:', 'ko' (black circles).

ExampleWorked example — plot a parabola▶
Example — y = x² for x in [−3, 3]
x = -3:0.1:3; y = x.^2; % element-wise power! plot(x, y); title('Parabola');
Note x.^2 with the dot — it squares each element of the vector. x^2 would error.

Topic 2 · Week 6
Subplots, 3-D & Log Scales
Beyond a single line on linear axes
Engineering data comes in many shapes. You may need several plots side by side (subplot), a surface over two variables (mesh), or axes that span many orders of magnitude (log scales). Picking the right plot type makes the data's story obvious.
Patternsubplot — a grid of plots▶
subplot(2,1,1); % 2 rows, 1 col, slot 1 (top) plot(x, sin(x)); subplot(2,1,2); % slot 2 (bottom) plot(x, cos(x));

subplot(rows, cols, n) selects the n-th cell of a grid so several plots share one figure window.

Definition3-D surfaces: meshgrid + mesh▶

To plot \(z = f(x,y)\) you first build a grid of (x,y) points with meshgrid, then mesh or surf:

[X, Y] = meshgrid(-2:0.2:2, -2:0.2:2); Z = X.^2 + Y.^2; mesh(X, Y, Z); % wireframe surface (a bowl)
PatternLogarithmic & polar plots▶
CommandAxes
semilogxx log, y linear
semilogyx linear, y log
loglogboth log
polarplotangle & radius (polar)
Why log scales? When data spans huge ranges (e.g. frequency response from 1 Hz to 1 MHz) a log axis compresses it so all the detail is visible.

Topic 3 · Week 6
GUIs & App Designer
Making interactive tools, not just scripts
A Graphical User Interface (GUI) lets a non-programmer use your tool with buttons, sliders and text boxes instead of typing code. MATLAB's App Designer is a drag-and-drop builder for exactly this. The key concept is the callback — code that runs when the user does something (clicks, drags).
DefinitionGUI components and callbacks▶
ComponentUsed for
ButtonTrigger an action
Edit fieldType a number/text
SliderChoose a value in a range
AxesDisplay a plot
LabelShow static or result text

A callback is a function attached to a component. When the user clicks a button, MATLAB runs that button's callback. This is "event-driven" programming.

PatternTwo ways to build a GUI▶
1
App Designer (appdesigner) — modern drag-and-drop; generates the code for you.
2
Programmatic — create components in code with uifigure, uibutton, uislider and set their callbacks.

Topic 4 · Week 7
Import & Export
Getting data in and results out
Real engineering data lives in files — sensor logs in .csv, photos in .jpg, spreadsheets in .xlsx. Knowing how to read these into MATLAB and write your results back out is essential. MATLAB offers high-level functions for common formats and a low-level interface for full control.
DefinitionHigh-level read/write functions▶
FormatReadWrite
Image (.jpg/.png)imreadimwrite
Table (.xlsx/.csv)readtablewritetable
Numeric matrixreadmatrixwritematrix
Cell datareadcellwritecell
Variables (.mat)loadsave
T = readtable('data.csv'); % read spreadsheet into a table img = imread('photo.jpg'); % read image into an array writematrix(results, 'out.csv');
PatternLow-level file access▶

For full control (e.g. custom text formats) use the low-level interface. The pattern is always open → process → close:

fid = fopen('data.txt', 'r'); % 'r'=read, 'w'=write while ~feof(fid) % until end-of-file line = fscanf(fid, '%f', 1); ... end fclose(fid); % ALWAYS close the file
Always fclose: a file you open but never close stays locked and can corrupt data. Pair every fopen with an fclose.

Topic 5 · Week 13
Transition to Python
The skills transfer — only the syntax changes
Once you understand variables, loops and functions, learning a second language is mostly learning new syntax. Python is the most popular language in engineering and data science. The concepts you learned in MATLAB map almost one-to-one — but watch the two big traps: Python uses 0-based indexing and indentation (not end) to mark code blocks.
DefinitionMATLAB vs Python — key differences▶
FeatureMATLABPython
File extension.m.py
Comment%#
End of statement; (optional)newline (no ;)
Block structureendindentation
First index10
Power^**
The #1 transition bug: in Python list[0] is the first element and list[1] is the second. In MATLAB those would be v(1) and v(2). Off-by-one errors are extremely common when switching.
DefinitionPython data structures▶
TypeSyntaxNotes
List[1, 2, 3]Ordered, changeable
Tuple(1, 2, 3)Ordered, fixed
Set{1, 2, 3}Unique, unordered
Dictionary{'a': 1}Key → value pairs
ExampleThe same loop in both languages▶
MATLAB
for i = 1:5 disp(i^2) end
Python
for i in range(1, 6): # range stops BEFORE 6 print(i**2) # indentation = the block
Libraries: Python gets MATLAB-like power from libraries — import numpy as np for arrays/maths, import matplotlib.pyplot as plt for plotting, and tkinter for GUIs.
