Scripts vs Command Window

A script is a plain text file ending in .m that contains a sequence of MATLAB commands. Running the script is identical to typing those commands one by one. Scripts share the workspace — variables they create persist after the script finishes.

Convention: Start every script with clc; clear; close all to clear the command window, delete old variables, and close old figures. This prevents bugs caused by leftover state from previous runs.

Output and Suppression

By default MATLAB prints the result of every statement. Add a semicolon ; to suppress output. Without it, every line prints — fine for debugging, messy in final code.

x = 5          % prints: x = 5
y = 10;        % silent — no output
z = x + y     % prints: z = 15

Comments

A percent sign % starts a comment. Everything after it on that line is ignored. Comments explain why, not what — the code itself shows what.

% This whole line is a comment
r = 5;      % radius in metres — inline comment

Order of Operations (PEMDAS/BODMAS)

MATLAB follows standard mathematical precedence:

  1. Parentheses () — innermost first
  2. Exponentiation ^ — right to left
  3. Unary minus -x
  4. Multiplication *, Division / — left to right
  5. Addition +, Subtraction - — left to right

Variable Naming Rules

  • Must start with a letter
  • Can contain letters, digits, and underscore _
  • Case-sensitive: Mass and mass are different variables
  • Cannot be a MATLAB keyword (for, if, end, etc.)
  • Avoid names that shadow built-in functions: don't name a variable sum, max, pi

Worked Example 2.1 — Kinematic calculation

A projectile launched at 25 m/s at 40°. Calculate horizontal and vertical velocity components.

clc; clear;
v0  = 25;          % m/s
deg = 40;          % degrees
rad = deg2rad(deg);

vx = v0 * cos(rad);
vy = v0 * sin(rad);
speed = sqrt(vx^2 + vy^2);

fprintf('vx    = %.4f m/s\n', vx);
fprintf('vy    = %.4f m/s\n', vy);
fprintf('speed = %.4f m/s\n', speed);
vx = 19.1511 m/s vy = 16.0697 m/s speed = 25.0000 m/s

Topic 3
Algorithm Design & Pseudocode
Problem decomposition, flowcharts, pseudocode, tracing, debugging

What is an Algorithm?

An algorithm is a finite, unambiguous sequence of instructions that solves a problem. Good algorithms are correct, efficient, readable, and handle edge cases.

The Engineering Problem-Solving Process

  1. Understand the problem — what are inputs, outputs, constraints?
  2. Work an example by hand — for small inputs, trace through manually
  3. Design the algorithm — flowchart or pseudocode
  4. Code it — translate to MATLAB
  5. Test with known cases — simple inputs where you know the answer
  6. Test edge cases — zero, negative, empty, very large inputs

Worked Example 3.1 — Counting odd numbers

Problem: Read N numbers from the user. Count how many are odd.

N = input('How many numbers? ');
count = 0;
for i = 1:N
    num = input('Enter number: ');
    if mod(num, 2) ~= 0
        count = count + 1;
    end
end
fprintf('Odd numbers: %d\n', count);

Topic 5
Loops — for, while, break, continue
Count-controlled, condition-controlled, nested, loop patterns

for Loop — Count-Controlled

Use when you know exactly how many iterations to run.

for variable = start:step:stop
    % body — runs for each value of variable
end
  • Default step is 1: for i = 1:10 → i = 1, 2, …, 10
  • Custom step: for k = 0:0.5:2 → k = 0, 0.5, 1.0, 1.5, 2.0
  • Negative step: for j = 10:-1:1

while Loop — Condition-Controlled

while condition
    % body — runs while condition is true
end

break and continue

KeywordEffect
breakExit the innermost loop immediately
continueSkip rest of current iteration, go to next

Topic 8
Vectorisation & Code Optimisation
Element-wise ops, pre-allocation, speed comparison

Why Vectorise?

MATLAB is built on highly optimised linear algebra libraries. Vectorised operations can be 10–100× faster than equivalent for loops.

Element-wise Operators

OperatorNameExampleResult
.*Element multiplyv .* [1 2 3][2 6 12]
./Element dividev ./ [2 3 4][1 1 1]
.^Element powerv .^ 2[4 9 16]

Pre-allocation

% BAD — array grows every iteration
result = [];
for i = 1:n
    result(end+1) = i^2;
end

% GOOD — allocate once
result = zeros(1, n);
for i = 1:n
    result(i) = i^2;
end

% BEST — fully vectorised
i = 1:n;
result = i.^2;

Topic 10
File I/O — Reading & Writing Data Files
fopen/fclose, fprintf, fscanf, fgetl, CSV, error handling

Why File I/O?

In engineering, data comes from sensors, instruments, simulations — not keyboard input. File I/O lets you load large datasets automatically, save results, and share data between programs.

The File Access Lifecycle

  1. Open — fid = fopen(filename, mode)
  2. Read/Write — use the file identifier fid
  3. Close — fclose(fid) — always, even if something went wrong

fopen Modes

ModeMeaningIf file doesn't exist
'r'Read onlyReturns fid = -1 (error)
'w'Write (overwrite)Creates new file
'a'AppendCreates new file

Always Check the File Opened Successfully

fid = fopen('data.txt', 'r');
if fid == -1
    error('Cannot open file: data.txt');
end
% ... read file ...
fclose(fid);

Writing Text Files with fprintf

fid = fopen('results.txt', 'w');
fprintf(fid, 'Results\n');
for i = 1:length(data)
    fprintf(fid, '%.4f\n', data(i));
end
fclose(fid);

Reading with fscanf

fid = fopen('temps.txt', 'r');
data = fscanf(fid, '%f');
fclose(fid);

CSV Files — Easiest Approach

% Reading a numeric CSV
data = readmatrix('sensors.csv');

% Writing a numeric CSV
writematrix(data, 'output.csv');
Worked Example 10.1 — Write and read data
%% Generate data
t   = (0:0.5:5)';
pos = 10*t - 0.5*9.81*t.^2;

%% Write to file

for i = 1:length(t)
    fprintf(fid, '%.1f %.4f\n', t(i), pos(i));

%% Read back
M = readmatrix('kinematics.txt');
