Section 01

Mental Model

Compression is the art of finding and eliminating redundancy. Every compressor is simultaneously a probability model of its source, and the fundamental connection between compression, prediction, and learning is not metaphor — it is theorem.

The Core Insight: Information = Surprise

When you see a common word like "the", you learn very little — it was expected. When you see a rare word like "zeugma", you learn a lot — it was unexpected. The quantity of information in an event is inversely related to its probability. Shannon formalised this as: the information content of event x with probability p(x) is exactly −log₂ p(x) bits. If you toss a fair coin and get heads (probability ½), you gain −log₂(½) = 1 bit of information. If a loaded coin lands heads with probability ¾, seeing heads only gives you −log₂(¾) ≈ 0.415 bits. This is the right measure because it satisfies the natural axiom that independent events' information adds: −log₂(p·q) = −log₂ p + (−log₂ q). Entropy H(X) is then just the average surprise — the expected information content over all outcomes. A source with low entropy is predictable, and predictable sources compress well.

The Two Families

Lossless compression

Exact reconstruction. Bounded below by Shannon entropy H(X) bits per symbol — no scheme can compress below this for i.i.d. sources without losing information. The LZ family achieves H(X) universally for any stationary ergodic source without knowing the source distribution in advance.

Lossy compression

Approximate reconstruction. Allows controlled distortion. Governed by the rate-distortion function R(D), which gives the minimum number of bits per symbol needed to achieve expected distortion ≤ D. For a Gaussian source with MSE distortion, R(D) = ½ log₂(σ²/D).

Three Deep Unities

Compression = Prediction

This is the deepest fact in the field. A code that assigns length −log₂ P(xₙ | x₁,...,xₙ₋₁) to each symbol is optimal under model P. The better your model predicts the next symbol, the shorter the code. This is not a metaphor: an arithmetic coder with an LLM's probability estimates is an information-theoretically optimal compressor for text drawn from the LLM's distribution. GPT-class language model perplexity is literally a compression rate in nats/token.

Compression = Learning

The Minimum Description Length (MDL) principle, due to Rissanen (1978), says: the best model for data is the model M that minimises K(M) + K(data | M) — the total description length. This is Occam's razor as a theorem. A model that overfits a dataset has high K(data | M) = 0 but high K(M); a model that underfits has high K(data | M). The sweet spot is the model that genuinely captures the regularities in the data. This unifies information theory with model selection, Bayesian inference, and statistical learning theory.

Compression = Measurement

Compressed sensing (Candès, Tao, Donoho, 2006) revealed that sparse signals can be acquired in compressed form directly — you don't acquire fully then compress, you compress during acquisition. A signal x with k nonzero entries in some basis needs only m = O(k log n/k) random linear measurements to recover exactly, far below the Nyquist rate of n measurements. The measurement process itself is a compression step, and recovery is a decompression step. This has been used in clinical MRI to reduce scan times by 4–8×.

The Information Hierarchy

H(X)
Shannon entropy

Average-case lower bound. A property of the probability distribution, not of specific strings. Achievable by optimal codes on long block lengths. Measured in bits per symbol.

K(x)
Kolmogorov complexity

Worst-case / instance-specific. Shortest program that outputs string x on a universal Turing machine. Uncomputable in general. For typical strings from distribution P, K(x) ≈ −log P(x) = H(P).

R(D)
Rate-distortion

Lossy regime. Minimum bits per symbol to reconstruct within distortion D. A function of the source distribution and the distortion measure. Generalises H(X) (which equals R(0) for lossless).

The Compression Pipeline

Every practical compressor — from gzip to neural codecs — instantiates the same two-layer architecture. The model estimates conditional probabilities P(xₙ | context). The coder takes those probabilities and converts them into bits. The coder can be near-perfect (arithmetic coding, ANS) so that the only remaining improvement is in the model. This decomposition explains why language models make better text compressors than gzip: the model is far better at predicting the next character in English text.

Discrete Probability & Stochastic Processes
prerequisite for everything
Probability spaces

A probability space is a triple (Ω, ℱ, P) where Ω is the sample space of all outcomes, ℱ is a σ-algebra of measurable events, and P : ℱ → [0,1] is a probability measure with P(Ω) = 1. For discrete spaces, ℱ = 2^Ω. A discrete random variable X is a function X : Ω → 𝒳 where 𝒳 is a countable set. The probability mass function (PMF) p(x) = P(X = x) satisfies p(x) ≥ 0 and ∑ₓ p(x) = 1. The PMF is the foundational object of lossless compression — the source model is a PMF over the alphabet 𝒳.

Key distributions in compression

Bernoulli(p): binary outcome; models a biased coin or a single bit in a binary source. H = −p log p − (1−p)log(1−p). Maximised at p = ½ where H = 1 bit.
Geometric(p): number of trials until first success; models run lengths in run-length encoding. Used to derive the Golomb codes optimal for geometric distributions.
Categorical: generalisation of Bernoulli to |𝒳| outcomes; the fundamental model for symbol coding.
Markov chain: Xₙ depends only on Xₙ₋₁ (first-order). Most practically useful source model. Transition matrix T where T[i,j] = P(Xₙ = j | Xₙ₋₁ = i). Context models in PPM, LZ77's implicit context, and JPEG's inter-block prediction are all Markov models.

Conditional probability and Bayes
P(A | B) = P(A ∩ B) / P(B) defined when P(B) > 0

Bayes' theorem: P(A|B) = P(B|A)P(A)/P(B). In compression, this is the update rule for adaptive models: after observing symbol x, update the probability model to reflect the new evidence. PPM does this explicitly; LZ-family does it implicitly by building a dictionary of observed strings.

Two events A, B are independent if P(A ∩ B) = P(A)P(B). If the symbols of a source are independent and identically distributed (i.i.d.), the optimal compressor assigns each symbol a code of length −log₂ p(x). Real sources are not i.i.d. — they have memory — which is why context models beat simple Huffman codes.

Expectation and moments
𝔼[f(X)] = ∑ₓ p(x) f(x)
Var(X) = 𝔼[(X − 𝔼X)²] = 𝔼[X²] − (𝔼X)²

Linearity of expectation: 𝔼[aX + bY] = a𝔼X + b𝔼Y (no independence needed). This is critical: the expected code length is 𝔼[l(X)] = ∑ₓ p(x)l(x), a linear functional of the code lengths.

Law of Large Numbers

For i.i.d. X₁, X₂,... with 𝔼X = μ: the sample average (1/n)∑ᵢXᵢ → μ in probability (weak LLN) and almost surely (strong LLN). This is the mathematical engine behind the Asymptotic Equipartition Property (Level 3): applying the LLN to Yᵢ = −log p(Xᵢ) gives −(1/n)log P(X₁,...,Xₙ) → H(X). Without the LLN, source coding theory has no foundation.

Markov chains and entropy rate

A stationary Markov chain has a unique stationary distribution π satisfying πᵀT = πᵀ. The entropy rate of a Markov chain is:

H(𝒳) = −∑ᵢ πᵢ ∑ⱼ T[i,j] log T[i,j]

This is the per-symbol entropy of the chain in steady state. LZ77 achieves this entropy rate for any ergodic Markov source, and in fact for any stationary ergodic source, without being told the chain's structure.

Why this matters for compression: the entropy rate H(𝒳) of your source model is the compression limit you're targeting. Everything in compression is about (a) estimating this value and (b) building a coder that approaches it.
2
Shannon Entropy
unlocks: optimal prefix codes, source coding
The self-information function

The information content (or surprisal) of observing outcome x with probability p(x) is defined as:

I(x) = −log₂ p(x)  bits

This is the unique function satisfying three natural axioms: (1) I(x) is continuous in p(x); (2) rarer events are more informative: I(x) is decreasing in p(x); (3) independent events' information adds: I(x,y) = I(x) + I(y) when x,y are independent, i.e. −log(pq) = −log p + (−log q). The logarithm is the unique function satisfying (3). Shannon proved these axioms uniquely determine I(x) = −log p(x) up to the choice of base (which sets the unit: base 2 = bits, base e = nats, base 10 = bans).

Entropy as expected surprise
H(X) = −∑_{x ∈ 𝒳} p(x) log₂ p(x) = 𝔼[−log₂ p(X)]

Convention: 0 log 0 = 0 (by continuity: x log x → 0 as x → 0). Entropy is the average information content of a random variable. For a fair coin: H = −½ log ½ − ½ log ½ = 1 bit. For a Bernoulli(p): H(p) = −p log p − (1−p)log(1−p), the binary entropy function h_b(p). H is maximised at p = ½.

Key properties of entropy
  • Non-negativity: H(X) ≥ 0. Since 0 ≤ p(x) ≤ 1, we have −log p(x) ≥ 0.
  • Maximised by uniform distribution: H(X) ≤ log₂|𝒳|, with equality iff p(x) = 1/|𝒳| for all x. Proof by Jensen's inequality: H(X) = 𝔼[log(1/p(X))] ≤ log 𝔼[1/p(X)] = log|𝒳|.
  • Concavity in p: H(λp + (1−λ)q) ≥ λH(p) + (1−λ)H(q). Mixing distributions increases entropy.
  • Chain rule: H(X,Y) = H(X) + H(Y|X) = H(Y) + H(X|Y). Information decomposes.
  • Subadditivity: H(X,Y) ≤ H(X) + H(Y), with equality iff X and Y are independent.
  • Conditioning reduces entropy: H(X|Y) ≤ H(X). Extra information never hurts.
KL divergence and the Gibbs inequality

The Kullback-Leibler divergence (relative entropy) from Q to P:

D_KL(P ∥ Q) = ∑_x P(x) log P(x)/Q(x)

The Gibbs inequality states D_KL(P ∥ Q) ≥ 0 with equality iff P = Q. Proof: −D_KL(P ∥ Q) = 𝔼_P[log(Q/P)] ≤ log 𝔼_P[Q/P] = log ∑ Q(x) = 0, by Jensen applied to the concave log function. This single fact implies all the other entropy inequalities. In compression: D_KL(P ∥ Q) is the number of extra bits per symbol you waste by using a Q-optimal code to encode P-distributed data. It is the compression cost of model mismatch.

H(P, Q) = H(P) + D_KL(P ∥ Q) cross-entropy = true entropy + KL penalty

This is why cross-entropy loss in machine learning training is a compression objective: minimising D_KL(data ∥ model) is exactly minimising the extra bits used by the model relative to the true distribution.

Mutual information
I(X;Y) = H(X) − H(X|Y) = H(Y) − H(Y|X) = H(X) + H(Y) − H(X,Y)

Mutual information measures how much knowing Y tells you about X. It is symmetric. It is zero iff X and Y are independent. It is the reduction in code length you achieve by using Y as side information when encoding X. In distributed source coding (Slepian-Wolf), mutual information appears as the rate savings when two correlated sources are jointly compressed.

Entropy rate for general processes

For a stationary stochastic process X₁, X₂,..., the entropy rate is:

H(𝒳) = lim_{n→∞} (1/n) H(X₁,...,Xₙ) = lim_{n→∞} H(Xₙ | X₁,...,Xₙ₋₁)

Both limits exist and are equal for stationary processes (Cesàro mean argument). The second limit is the more practically useful: it is the per-symbol entropy given all previous context — the fundamental quantity an arithmetic coder with a perfect model achieves.

Differential entropy for continuous sources

For a continuous random variable X with density f(x), the differential entropy is:

h(X) = −∫ f(x) log f(x) dx

Unlike discrete entropy, h(X) can be negative and is coordinate-system-dependent. But differences of differential entropies (mutual information, KL divergence) remain invariant. For a Gaussian X ~ 𝒩(μ, σ²): h(X) = ½ log(2πeσ²). The Gaussian maximises differential entropy among all distributions with fixed variance — this is why Gaussian sources appear as worst cases in rate-distortion analysis.

Worked example: A fair die has H = log₂ 6 ≈ 2.585 bits. A loaded die with P(1) = ½, P(2)=...=P(6) = 1/10 has H = −½·1 − 5·(1/10)·log₂(1/10) ≈ 2.16 bits. The loaded die is more predictable, so it can be compressed more.

Source Coding Theorem & Prefix Codes
unlocks: Huffman, arithmetic coding
What is a code?

A binary code for source 𝒳 is a function C : 𝒳 → {0,1}*. It is uniquely decodable if C(x₁)C(x₂)... can be parsed back into x₁,x₂,... unambiguously. It is a prefix code (instantaneous code) if no codeword is a prefix of another — meaning you can decode symbol by symbol without lookahead. All prefix codes are uniquely decodable, but not vice versa. In practice, prefix codes are preferred for their simplicity.

Kraft-McMillan inequality

For a uniquely decodable binary code with codeword lengths l₁, l₂,..., l_|𝒳|:

∑_{x ∈ 𝒳} 2^{−lₓ} ≤ 1

Proof sketch: Consider the K^n-fold extension. Count the number of sequences of length m in the tree of all binary strings. Uniquely decodable codes cannot have more length-m sequences than there are binary strings of length m. Taking the n-th root and n→∞ gives the result. Crucially, the inequality is also sufficient: if lengths {lₓ} satisfy it, a prefix code with those lengths exists (by the prefix tree construction).

Optimal code lengths and Shannon's theorem

Minimise expected length L̄ = ∑ p(x)l(x) subject to the Kraft inequality. The Lagrangian optimum is l*(x) = −log₂ p(x). Since these aren't always integers, we round up: lₓ = ⌈−log₂ p(x)⌉. The resulting code satisfies:

H(X) ≤ L̄ < H(X) + 1

The gap of up to 1 bit per symbol is the integer constraint inefficiency. For long blocks of n symbols, the gap shrinks to 1/n — approaching H(X) per symbol. This is why arithmetic coding (which achieves H(X) exactly, without integer constraints) matters.

Huffman coding: algorithm and optimality

The Huffman algorithm greedily builds the optimal prefix code:

  1. Create a leaf node for each symbol x with weight p(x).
  2. Repeatedly merge the two nodes of lowest weight into an internal node with weight equal to their sum.
  3. Repeat until one root node remains. The resulting binary tree's leaf depths are the code lengths.
Example: Symbols {A, B, C, D} with probabilities {0.5, 0.25, 0.125, 0.125}. Merge C+D → node(0.25); merge this with B → node(0.5); merge with A → root(1.0). Lengths: A=1, B=2, C=3, D=3. L̄ = 0.5·1 + 0.25·2 + 0.125·3 + 0.125·3 = 1.75 bits. H = −(0.5 log 0.5 + 0.25 log 0.25 + 2·0.125 log 0.125) = 1.75 bits exactly. This is a case where H is achieved — because all probabilities are powers of ½.

Optimality proof: By exchange argument. If the optimal code does not have the property that the two least-probable symbols have the longest (equal-length) codewords as siblings, then swapping them to achieve this property cannot increase L̄. The inductive step then shows Huffman's greedy choice is globally optimal among all prefix codes.

The AEP and the source coding theorem

The Asymptotic Equipartition Property (AEP) is the LLN applied to the log-probability. For i.i.d. X₁,...,Xₙ from distribution P:

−(1/n) log P(X₁,...,Xₙ) → H(X)  in probability

This means for large n, almost all of the probability mass is concentrated on the set 𝒯_ε of typical sequences: those satisfying |−(1/n) log P(x^n) − H| < ε. The typical set has two key properties:

  • |𝒯_ε| ≤ 2^{n(H+ε)} — there are at most 2^{n(H+ε)} typical sequences.
  • P(X^n ∈ 𝒯_ε) → 1 as n → ∞.

The source coding theorem follows: to encode n-symbol blocks with vanishing error probability, nH(X) bits suffice (encode only the 2^{nH} typical sequences, declare error otherwise). Below nH(X) bits: no matter what code you use, error probability approaches 1. This is Shannon's fundamental compression limit.

Common confusion: H(X) bounds the average description length per symbol for i.i.d. sources. For sources with memory (Markov chains, general stationary processes), the bound is the entropy rate H(𝒳) = lim (1/n)H(X₁,...,Xₙ), which is ≤ H(X₁). A context model exploiting the memory achieves H(𝒳) while a memoryless code achieves only H(X₁) — the difference is the compression gain from modelling context.

Arithmetic Coding & ANS
unlocks: near-entropy coding in practice
The integer constraint problem

Huffman coding wastes up to 1 bit per symbol due to integer codeword lengths. For a symbol with p(x) = 0.99, the optimal code length is −log₂(0.99) ≈ 0.0145 bits, but Huffman must assign 1 bit. The overhead is a factor of 1/0.0145 ≈ 69! Arithmetic coding solves this by never assigning individual codewords to individual symbols — instead, the entire message is encoded as a single number.

The interval subdivision procedure

Arithmetic coding maps a sequence x₁,...,xₙ to a sub-interval of [0, 1). Begin with the full interval [0, 1). For each symbol xᵢ, subdivide the current interval [l, h) into sub-intervals proportional to the probabilities of each symbol. Keep the sub-interval corresponding to xᵢ. After n symbols, you have a final interval [l*, h*) with width h* − l* = ∏ᵢ p(xᵢ).

Step-by-step example: Alphabet {A, B} with P(A)=0.7, P(B)=0.3. Encode "AB". Start with [0, 1). Symbol A → keep [0, 0.7). Symbol B → subdivide [0, 0.7) into [0, 0.49) for A and [0.49, 0.7) for B → keep [0.49, 0.7). Any binary fraction in [0.49, 0.7) encodes "AB". The interval width is 0.21 = 0.7·0.3. Code length = ⌈−log₂(0.21)⌉ + 1 = 3 bits (e.g., 0.5 = 0.1₂ in binary). For comparison, P("AB") = 0.21, so the optimal length is 2.25 bits — arithmetic coding achieves this within 1 bit overhead for the entire message, not per symbol.
Decoding and the key property

To decode: given the transmitted binary number z ∈ [0,1), find which sub-interval it falls in at each step. This uniquely determines each symbol xᵢ. The crucial property: the decoder can work incrementally — output each symbol as soon as the sub-interval is narrow enough to commit.

Bits needed = ⌈−log₂ P(x₁,...,xₙ)⌉ + 1

This achieves within 1 bit of the theoretical optimum −log₂ P(x^n) for the entire message, regardless of message length n. Per-symbol overhead → 0 as n → ∞. Unlike Huffman, arithmetic coding achieves entropy exactly in the limit.

Finite precision and the carry problem

The real-valued interval narrows exponentially. A practical implementation must use finite-precision integers. The Rissanen-Langdon (1979) solution: output bits as soon as the most significant bit of l and h agree, then rescale. This "bit-at-a-time" output is possible because once the high and low bounds agree on a bit, that bit is determined regardless of future symbols. The "carry" case (l and h straddle 0.5 without agreeing) requires the "follow bits" technique: buffer the uncertain bit and output it + its correction when the uncertainty resolves. Modern range coding (Pasco, Martin) implements this cleanly with integer arithmetic.

The model-coder separation

Arithmetic coding is a universal coder: given any probability model P(xₙ | x₁,...,xₙ₋₁), it achieves within 1 bit of the entropy of that model over the entire message. This means you can independently improve the model (PPM, neural networks, transformers) and the coder separately. JPEG2000 uses an arithmetic coder (MQ-coder) with a context model. H.264 uses CABAC (Context-Adaptive Binary Arithmetic Coding). All modern learned image codecs use arithmetic coding with a learned entropy model.

ANS: Asymmetric Numeral Systems (Duda, 2009)

ANS achieves the same compression quality as arithmetic coding but at near-Huffman speed, enabling hardware SIMD implementations. The key idea: encode a stream of symbols into a single growing integer. Define a state s ∈ {L,...,2L−1} for some L. The encoder maps (s, x) → s', where the new state s' encodes the symbol. The decoder reverses this bijection. ANS is a bijection between sequences over a weighted alphabet and the natural numbers, with the bit output rate equal to the Shannon information content of each symbol on average.

Two variants: rANS (ranged ANS) is simpler and easier to analyse; tANS (table ANS, marketed as Finite State Entropy / FSE) uses lookup tables for speed. Both power all major modern codecs: zstd, LZMA2, JPEG XL, AV1, Apple's lossless audio codec.

Linear Algebra for Transform Coding
unlocks: JPEG, SVD compression, image coding
Why transform coding?

A natural image signal has strong spatial correlation: adjacent pixels tend to have similar values. Encoding them independently ignores this redundancy. The key insight of transform coding: apply a linear transformation T to the signal x to get coefficients c = Tx. If T is chosen well, the coefficients cᵢ will be (approximately) uncorrelated. Uncorrelated components can be quantized and encoded independently — and their independent bits sum to the total bitrate — whereas correlated components waste bits encoding redundancy.

Orthonormal bases

A set of vectors {e₁,...,eₙ} in ℝⁿ forms an orthonormal basis if ‹eᵢ, eⱼ› = δᵢⱼ. Any signal x decomposes uniquely as x = ∑ᵢ cᵢeᵢ where cᵢ = ‹x, eᵢ›. The transformation T with rows {eᵢᵀ} is orthogonal: TᵀT = TTᵀ = I. Orthogonality implies: (1) no distortion in the transform domain — ‖Tx‖₂ = ‖x‖₂ (Parseval/isometry); (2) coefficients can be independently encoded; (3) the transform is its own inverse Tᵀ = T⁻¹.

Singular Value Decomposition

Every m×n real matrix A has an SVD: A = UΣVᵀ where U ∈ ℝ^{m×m} and V ∈ ℝ^{n×n} are orthogonal, and Σ ∈ ℝ^{m×n} is diagonal with non-negative entries σ₁ ≥ σ₂ ≥ ... ≥ 0 (the singular values). The columns of U (V) are left (right) singular vectors. Geometrically, every linear map is a rotation, followed by scaling along orthogonal axes, followed by another rotation.

A = ∑ᵢ σᵢ uᵢ vᵢᵀ outer product expansion; rank-1 summands

The Eckart-Young theorem (1936): among all rank-k matrices B, the truncated SVD Aₖ = ∑_{i=1}^k σᵢ uᵢvᵢᵀ minimises ‖A − B‖_F and ‖A − B‖₂. This is the theoretical basis of image compression by matrix approximation: storing Aₖ requires k(m+n+1) numbers instead of mn. For a 1000×1000 image with rank-50 approximation: 1,000,000 → 101,000 numbers.

PCA and the KL transform

Given a zero-mean random vector X ∈ ℝⁿ with covariance Σ_X = 𝔼[XXᵀ], the eigendecomposition Σ_X = QΛQᵀ defines the Karhunen-Loève Transform (KLT): apply the change of basis Y = QᵀX. Then:

Σ_Y = 𝔼[YYᵀ] = Qᵀ Σ_X Q = Λ diagonal! — the Yᵢ are uncorrelated

The KLT is the uniquely optimal orthogonal transform for data compression in two senses: (1) it maximises energy compaction — any k coefficients of the KLT capture more variance than any k coefficients in any other orthonormal basis; (2) for Gaussian sources, the KLT also minimises mean-squared reconstruction error for a given bitrate (water-filling optimum). The problem: the KLT requires knowing Σ_X (the source statistics). For images, Σ_X varies by image content. The DCT approximates the KLT for the class of stationary first-order Markov processes that model natural images well.

The coding gain of transform coding

The coding gain G_TC measures how much better transform coding is than scalar PCM (pulse-code modulation, i.e., encoding each sample directly):

G_TC = (1/n ∑ᵢ σᵢ²) / (∏ᵢ σᵢ²)^{1/n} = arithmetic mean / geometric mean of variances

This is always ≥ 1 (AM-GM inequality), with equality only when all variances are equal (i.e., when the signal is already uncorrelated). A large coding gain means the transform significantly decorrelates the signal — more energy is concentrated in fewer coefficients. The DCT achieves coding gains of 8–15 dB on natural images at typical compression ratios.

Fourier Analysis and the Discrete Cosine Transform
unlocks: JPEG, audio coding, spectral methods
From continuous Fourier to DFT

The continuous Fourier transform F(ω) = ∫ f(t)e^{−jωt}dt decomposes a signal into complex sinusoids. For digital signals sampled at N points, the Discrete Fourier Transform (DFT) is:

X[k] = ∑_{n=0}^{N−1} x[n] e^{−j2πkn/N} k = 0,...,N−1

The DFT basis functions are complex exponentials of frequencies k/N cycles per sample. Parseval's theorem: ∑|x[n]|² = (1/N)∑|X[k]|² — total energy is conserved across the transform.

The boundary discontinuity problem

The DFT implicitly assumes the signal is periodic: it treats x[0],...,x[N−1] as one period of an infinite periodic signal. For most real signals, x[0] ≠ x[N−1], creating an artificial discontinuity at the boundary. Discontinuities in time create high-frequency "ringing" artefacts in the frequency domain (Gibbs phenomenon). For image blocks, this produces blocking artefacts visible in JPEG-compressed images — especially at low bitrates.

The DCT-II and symmetric extension

The DCT-II solves the boundary problem by treating the N-point signal as one period of a signal with even symmetry about both boundaries — effectively mirroring the signal at both ends. The resulting basis functions are cosines:

X[k] = ∑_{n=0}^{N−1} x[n] cos(π(2n+1)k / 2N) k = 0,...,N−1

The implicit symmetric extension creates a smooth periodic signal (no discontinuity at the boundary), dramatically reducing high-frequency artefacts. The DC component X[0] = ∑x[n] (total energy); higher coefficients capture spatial frequencies. The four DCT variants (I, II, III, IV) differ in the symmetry assumptions; DCT-II (and its inverse DCT-III) is standard in image/video coding.

DCT-II nearly diagonalises natural image covariance

A first-order autoregressive (AR(1)) Markov process has covariance Σ[i,j] = σ²ρ^|i−j| for correlation coefficient ρ. The Jayant-Noll (1984) result (building on Ahmed-Natarajan-Rao, 1974): as N → ∞, the KLT of an AR(1) process converges to the DCT-II. For natural images, ρ ≈ 0.9–0.95 horizontally and vertically. The DCT is therefore an excellent approximation to the optimal KLT, without requiring knowledge of the image statistics. This is the theoretical justification for its use in JPEG.

The 8×8 block DCT in JPEG

JPEG applies the 2D DCT-II independently to non-overlapping 8×8 blocks of pixel values. The 2D DCT separates into row and column 1D DCTs (exploiting separability of the cosine basis). Each 8×8 block produces 64 frequency coefficients. Typical natural image blocks have most energy in the top-left (DC + low spatial frequencies). The JPEG quantization matrix divides each coefficient by a step size Q_k — larger Q_k for high frequencies (which contribute less to perceptual quality). After quantization, most high-frequency coefficients become zero. Zig-zag ordering scans the 8×8 block diagonally (increasing frequency) to cluster zeros together for run-length coding.

JPEG quality factor Q → Q_k = ⌊(Q_base_k · S + 50) / 100⌋ S = 5000/Q for Q < 50, S = 200−2Q otherwise
Fast DCT and computational cost

Naïve DCT costs O(N²) multiplications. The fast DCT (Chen-Wang-Itoh, 1977) exploits the recursive structure of the cosine basis to achieve O(N log N) using N/2 recursion stages. For 8×8 blocks: 8 points reduces to two 4-point DCTs, each to two 2-point. The result: 8-point DCT needs 11 multiplications and 29 additions (vs 64 multiplications naïvely). This makes JPEG computationally practical.

Wavelet Theory & Multiresolution Analysis
unlocks: JPEG2000, scalable coding, denoising
The problem with DCT

The DCT operates on fixed-size blocks, making it a global frequency representation within each block. An edge in the middle of a block — a sharp transition between two regions — produces large coefficients across all 64 frequencies. The block cannot represent the edge compactly. Furthermore, the block boundaries create discontinuities that cause "blocking artefacts" at low bitrates. What's needed is a representation that is simultaneously localised in both space and frequency.

Time-frequency uncertainty principle

The Heisenberg uncertainty principle for signal analysis states: a signal cannot be simultaneously arbitrarily concentrated in time (space) and in frequency. Specifically, Δt · Δω ≥ ½ for any signal, where Δt and Δω are the standard deviations of the signal's energy in time and frequency domains. Fourier transforms have perfect frequency localisation (infinite time spread). DCT has frequency localisation within blocks (finite but fixed time spread). Wavelets make a different tradeoff: fine temporal resolution for high-frequency components, coarse temporal resolution for low-frequency components — matching how natural signals actually behave (rapid oscillations are localised; slow trends are global).

The Haar wavelet: construction from scratch

The simplest wavelet is the Haar wavelet ψ(t) = 1 on [0, ½), −1 on [½, 1), 0 elsewhere. Apply it to a 4-sample signal x = [4, 6, 2, 2]:

  • Level-1 averages (scaling coefficients): [(4+6)/2, (2+2)/2] = [5, 2]
  • Level-1 differences (wavelet coefficients): [(4−6)/2, (2−2)/2] = [−1, 0]
  • Level-2 (apply again to averages): average = [(5+2)/2] = [3.5]; difference = [(5−2)/2] = [1.5]

Output: [3.5, 1.5, −1, 0]. The single DC coefficient (3.5) captures the average level. The level-2 wavelet coefficient (1.5) captures the coarse structure. The level-1 coefficients capture fine detail. Most energy is in low coefficients. Thresholding the small wavelet coefficients to zero gives compression without blocking artefacts — the Haar transform implicitly separates the signal at multiple scales.

What makes a valid wavelet

A function ψ(t) is a wavelet if: (1) it has zero mean: ∫ψ(t)dt = 0 (admissibility condition, required for invertibility); (2) it has finite energy: ∫|ψ(t)|² dt < ∞. The admissibility condition in frequency domain: ∫|Ψ(ω)|²/|ω| dω < ∞, which implies Ψ(0) = 0, i.e., ψ integrates to zero. The family {ψ_{s,τ}(t) = (1/√s) ψ((t−τ)/s)} of scaled and translated wavelets forms an analysing family. For discrete transforms, dyadic scaling (s = 2^j) and integer translations give the discrete wavelet transform (DWT).

Vanishing moments and compression

A wavelet has K vanishing moments if ∫ t^k ψ(t) dt = 0 for k = 0, 1,..., K−1. This means the wavelet is orthogonal to polynomials of degree < K. Why does this matter for compression? If a signal is locally polynomial of degree < K in some region, the wavelet coefficients in that region will be near zero — the wavelet is "blind" to smooth regions. Only at edges or discontinuities (where the polynomial approximation fails) will wavelet coefficients be large. More vanishing moments → sparser representations → better compression for smooth signals with isolated features (exactly the structure of natural images and audio).

Multiresolution Analysis (Mallat, 1989)

The MRA framework characterises which functions can serve as scaling functions φ (for averages) and wavelets ψ (for differences). A sequence of nested subspaces V_j ⊂ V_{j+1} ⊂ L²(ℝ) with basis {φ(t−n)} for V₀, {φ(2t−n)} for V₁, ... satisfies:

φ(t) = √2 ∑_n h[n] φ(2t − n) two-scale equation — scaling function as filtered/downsampled version of itself
ψ(t) = √2 ∑_n g[n] φ(2t − n) wavelet defined via high-pass filter g

The sequences h (low-pass) and g (high-pass) are the quadrature mirror filters and must satisfy H(ω)H*(ω) + G(ω)G*(ω) = 1 for perfect reconstruction. The DWT is implemented by alternating filtering + downsampling (Mallat's pyramidal algorithm), at cost O(N) — faster than the O(N log N) FFT.

Daubechies wavelets and JPEG2000

The Haar wavelet has only 1 vanishing moment and poor smoothness (it's discontinuous). Daubechies wavelets (1988) are the unique compactly-supported orthogonal wavelets with maximum vanishing moments for a given support length. DbN has N vanishing moments, support [0, 2N−1], and regularity ~0.2N. For compression, more vanishing moments → sparser coefficients for smooth signals. The tradeoff: longer filters introduce more computational overhead and ringing near image boundaries.

JPEG2000 uses the CDF 9/7 biorthogonal wavelet (Cohen-Daubechies-Feauveau) for lossy coding: 9-tap analysis filter and 7-tap synthesis filter. Biorthogonal wavelets allow the analysis and synthesis filters to be different, enabling better frequency selectivity and smoother reconstruction. The lifting scheme (Sweldens, 1996) provides an efficient O(N) implementation by decomposing the filter bank into a series of prediction and update steps, also enabling integer-to-integer wavelet transforms for lossless coding.

Why wavelets beat DCT at low bitrates: At high compression, DCT quantizes high-frequency coefficients to zero, creating blocking artefacts at block boundaries. Wavelet coefficients represent edges across scales; when small coefficients are zeroed, the reconstruction remains smooth everywhere except at the preserved edge locations. JPEG2000 images at 0.1 bits/pixel look blurry-but-smooth; JPEG images look blocky. At high bitrates (>1 bpp), the difference is negligible.

Rate-Distortion Theory
unlocks: lossy codec design, neural compression
The problem and the definition

Lossless compression is impossible below H(X). But if we allow a small error in reconstruction, how many bits per symbol do we actually need? This is the question rate-distortion theory answers. Given source X, a reconstruction alphabet X̂, and a distortion measure d(x, x̂) ≥ 0 (e.g., MSE d(x,x̂) = (x−x̂)², Hamming distance d(x,x̂) = 𝟏[x≠x̂]), the rate-distortion function is:

R(D) = min_{p(x̂|x): 𝔼[d(X,X̂)]≤D} I(X; X̂)

This is a constrained optimisation over all conditional distributions p(x̂|x) (the "test channel" or "reconstruction channel"). R(D) is the minimum mutual information between source and reconstruction achievable at distortion ≤ D. Shannon's channel coding theorem (in reverse) shows this rate is achievable with random codes at block length n → ∞, and nothing below it is achievable.

Properties of R(D)
  • Non-increasing: R(D₁) ≥ R(D₂) for D₁ ≤ D₂. More distortion allowed → fewer bits needed.
  • Convex: R(D) is convex in D. The curve is smooth (for continuous sources).
  • R(0) = H(X) for discrete sources: zero distortion requires lossless compression.
  • R(D) = 0 for D ≥ D_max where D_max = min_{x̂} 𝔼[d(X, x̂)]: at some distortion level, we can replace everything with a single reconstruction symbol for zero bits.
Gaussian source: exact derivation

For X ~ 𝒩(0, σ²) with MSE distortion d(x,x̂) = (x−x̂)²:

Step 1: Use the Lagrangian relaxation: minimise I(X;X̂) + λ𝔼[(X−X̂)²] over p(x̂|x). By the entropy power inequality, for fixed 𝔼[(X−X̂)²] = D, the minimising channel is Gaussian: X̂ = X + Z where Z ~ 𝒩(0, D') is independent noise, but we want the reverse-channel view.

Step 2: The optimal test channel: X = X̂ + Z where X̂ ~ 𝒩(0, σ²−D) and Z ~ 𝒩(0, D), independent. Then 𝔼[(X−X̂)²] = D and I(X;X̂) = H(X) − H(X|X̂) = ½log(σ²/D).

R(D) = max(0, ½ log₂(σ²/D)) bits/sample, for 0 ≤ D ≤ σ²

At D = σ²: R = 0 (just output zero every time). At D → 0: R → ∞ (perfect reconstruction needs infinite bits). For every 6 dB increase in PSNR (factor of 4 reduction in MSE), 1 extra bit per sample is needed.

Water-filling for vector Gaussian sources

For a vector source X ∈ ℝⁿ with independent Gaussian components Xᵢ ~ 𝒩(0, σᵢ²), the total rate-distortion function decomposes:

R(D) = ∑ᵢ max(0, ½ log₂(σᵢ²/λ))
D = ∑ᵢ min(σᵢ², λ)

The Lagrange multiplier λ (the "water level") is determined by the distortion constraint. The optimal bit allocation assigns bits to components with high variance (interesting signal) and none to components with low variance (below λ, where reconstructing with the mean is already near-optimal). This is the water-filling analogy: pour "water" (bits) into a "landscape" (variance profile) until the water level reaches λ.

This is why transform coding approximates R(D): the KLT diagonalises the covariance (makes components independent), and water-filling then optimally allocates bits. DCT + quantization + variable-length coding implements this approximately.

The Shannon lower bound

For any continuous source with differential entropy h(X) and MSE distortion:

R(D) ≥ h(X) − ½ log₂(2πeD)

This is tight for Gaussian sources. For non-Gaussian sources, the true R(D) lies above this bound. The gap between a source's true R(D) and the Gaussian R(D) (with the same variance) measures how much the non-Gaussianity helps compression. Natural image distributions (heavy-tailed, non-Gaussian) have lower true R(D) than the Gaussian bound — they are more compressible than a Gaussian with the same energy.

The Blahut-Arimoto algorithm

For arbitrary discrete sources, R(D) is computed iteratively via alternating optimisation:

Initialize p(x̂) = uniform Repeat until convergence: p*(x̂|x) ∝ p(x̂) exp(−λ d(x, x̂)) [optimal reconstruction channel] p(x̂) = ∑_x p(x) p*(x̂|x) [marginal update]

This double-loop converges to the globally optimal p(x̂|x) achieving R(D) at rate λ. It is the information-theoretic analogue of the EM algorithm and the ancestor of all iterative methods in compression optimisation, including the Blahut algorithm for channel capacity.

Kolmogorov Complexity & Algorithmic Information Theory
unlocks: MDL, universal learning, NCD
Motivation: the limits of Shannon theory

Shannon entropy H(X) is a property of a probability distribution. It tells us the average complexity of a source, assuming we know (or have estimated) the source distribution. But what about a single specific string? The string "01010101..." has very low complexity — it follows a simple rule — but its Shannon entropy (from a uniform source model) would be 1 bit/symbol. Shannon theory cannot capture the regularities in individual strings; it can only describe ensembles.

Kolmogorov complexity: the definition

Fix a universal Turing machine U (e.g., a Python interpreter). The Kolmogorov complexity of a binary string x is:

K(x) = min{|p| : U(p) = x}

the length of the shortest program p (in bits) such that U running on p halts and outputs x. For "01010101..." repeated 1000 times: K(x) ≈ log₂(1000) + C (length of "print '01'×500") — very short. For a random 1000-bit string: K(x) ≈ 1000 (no short description exists).

Invariance theorem: for any two universal TMs U and V, |K_U(x) − K_V(x)| ≤ c_{U,V}, a constant depending only on the machines, not on x. So K(x) is well-defined up to an additive constant. The choice of programming language doesn't matter for long strings.

Incompressibility and random strings

For any n and any constant c, the number of strings of length n with K(x) < n − c is at most 2^{n−c} (since there are only 2^{n−c} programs of length < n−c). Therefore at most 2^{n−c}/2^n = 2^{−c} fraction of length-n strings have K(x) < n−c. For c=10: fewer than 1/1024 of all strings are compressible by 10 bits. Most strings are incompressible — they cannot be described more briefly than by writing them out. This is why compressors fail on already-compressed or random data: there is no shorter description.

Uncomputability

K(x) is not computable. Proof by contradiction: if K were computable, then for any n we could enumerate all strings with K(x) ≤ n and eventually output one with K(x) > n (Berry's paradox / halting problem). Formally, the set {(x, k) : K(x) ≤ k} is recursively enumerable (you can semi-decide "K(x) ≤ k" by running all programs of length ≤ k) but not recursive. No program can, in general, determine the shortest description of its input. In practice: all compressors are upper bounds on K(x), not the true K(x). Better compressors = tighter upper bounds.

The two-part code and MDL

The two-part code MDL (Rissanen, 1978) gives a practical operationalisation of Kolmogorov complexity for model selection. To transmit data x using model M, you need two parts: (1) the model M itself, and (2) the data given the model. Total code length:

L(M, x) = L(M) + L(x | M)

The best model is the one minimising L(M, x). A complex model M (long L(M)) can reduce L(x|M) to nearly zero (by memorising x), but the model description itself becomes long. A simple model M has short L(M) but poor fit. MDL selects the model at the sweet spot — this is Occam's razor formalised. MDL is equivalent to Bayesian model selection with a universal prior when L(M) = −log P(M) (description length = negative log prior probability).

Universal probability and Solomonoff induction

Define the universal probability m(x) = ∑_{p: U(p)=x} 2^{−|p|} — the probability of outputting x by a random program. By the coding theorem, log m(x) ≈ −K(x). This is the dominant semimeasure: for any computable probability P over strings, m(x) ≥ c_P · P(x) for all x (where c_P depends only on P). This means m(x) assigns higher probability to any string that any computable distribution assigns probability to.

Solomonoff induction uses m(x) as a universal prior for prediction: given x₁,...,xₙ, the predicted distribution for xₙ₊₁ is m(xₙ₊₁ | x₁,...,xₙ) ∝ m(x₁,...,xₙ₊₁). This is the theoretically optimal Bayesian predictor — it dominates every computable prediction strategy and achieves the minimum expected prediction error. It is uncomputable, but LLMs can be seen as computable approximations to it.

Normalised Compression Distance (NCD)

The NCD gives a computable approximation to the algorithmic mutual information between two strings x and y:

NCD(x, y) = (K(xy) − min(K(x), K(y))) / max(K(x), K(y))

where K(xy) is the complexity of the concatenation. Approximating K with a real compressor (gzip, bzip2, LZMA): NCD(x,y) ≈ (C(xy) − min(C(x), C(y))) / max(C(x), C(y)) where C is the compressed length. NCD ∈ [0, 1+ε]: 0 means x is as simple as possible given y (they are "the same"); 1 means they share no compressible structure. NCD has been used for phylogenetic analysis (clustering species by their genomic sequences), plagiarism detection, and classifying music by genre. All without domain knowledge — the compressor discovers the structure.

Compressed Sensing & Sparse Recovery
unlocks: sub-Nyquist acquisition, MRI, CS-RADAR
The sub-Nyquist puzzle

Classical sampling theory (Nyquist-Shannon): to reconstruct a signal with bandwidth W, sample at rate ≥ 2W. This is tight — no recovery is possible below 2W for arbitrary bandlimited signals. But what if the signal has additional structure? Specifically: what if it is sparse — has only k nonzero components in some basis — where k ≪ N? Compressed sensing (Candès, Romberg, Tao, 2006; Donoho, 2006) shows that such signals can be recovered from m = O(k log(N/k)) generic linear measurements — far fewer than N.

The ℓ₀ problem and its NP-hardness

Given y = Ax where x ∈ ℝⁿ is k-sparse and A ∈ ℝ^{m×n} with m < n, the natural recovery formulation is:

min ‖x‖₀ subject to Ax = y ‖x‖₀ = number of nonzeros

This is NP-hard in general (equivalent to subset selection). However, Candès, Tao, and Donoho independently showed that under conditions on A, the convex relaxation achieves the same solution:

min ‖x‖₁ subject to Ax = y Basis Pursuit — a linear program
The ℓ₁ geometry: why convex relaxation works

Consider minimising ‖x‖_p subject to Ax = y. The unit ℓ_p balls have very different geometry. The ℓ₂ ball is a sphere — intersecting it with a hyperplane Ax = y generically gives a solution with no zero components. The ℓ₁ ball is a cross-polytope (diamond in 2D, octahedron in 3D) — its extreme points are the coordinate vectors {±eᵢ}. When you inflate the ℓ₁ ball until it touches the constraint set Ax = y, it almost always hits a vertex or edge of the polytope, which corresponds to a sparse solution. ℓ₁ minimisation is the unique convex program that promotes sparsity in this geometric sense.

The Restricted Isometry Property

Matrix A ∈ ℝ^{m×n} satisfies RIP(k, δ) if for all k-sparse vectors x:

(1 − δ)‖x‖₂² ≤ ‖Ax‖₂² ≤ (1 + δ)‖x‖₂²

RIP says A preserves the ℓ₂ norm of all k-sparse vectors — it acts as an approximate isometry on the set of k-sparse vectors. If A satisfies RIP(2k, δ) with δ < √2 − 1 ≈ 0.414, then Basis Pursuit exactly recovers every k-sparse x from y = Ax.

Proof sketch: Let x* be the true sparse signal and x̂ be the BP solution. Define h = x̂ − x*. Since both satisfy Ax̂ = y = Ax*, we have Ah = 0. Write h = h_S + h_{S^c} where S is the support of x*. Since x̂ is ℓ₁ minimal: ‖x*_S + h_S‖₁ + ‖h_{S^c}‖₁ = ‖x̂‖₁ ≤ ‖x*‖₁ = ‖x*_S‖₁, so ‖h_{S^c}‖₁ ≤ ‖h_S‖₁. The RIP condition then forces ‖h‖₂ to be small. Full details require the cone constraint and RIP algebra — see Candès 2008.

Random matrices satisfy RIP

A random matrix A with entries drawn i.i.d. from 𝒩(0, 1/m) satisfies RIP(k, δ) with high probability 1 − e^{−cm} when m ≥ CK(δ)k log(n/k). The proof uses the Johnson-Lindenstrauss lemma (a random projection approximately preserves the distance between any finite set of points) combined with a covering argument over the set of k-sparse unit vectors. Gaussian, Bernoulli (±1/√m), and subgaussian random matrices all work. Partial Fourier matrices (random rows of the DFT matrix) also satisfy RIP — directly enabling CS-MRI (subsample k-space).

Algorithms for sparse recovery
  • Basis Pursuit (BP): Exact ℓ₁ minimisation via LP. Optimal in theory; O(n³) computational cost — slow for large n.
  • LASSO: min ‖y − Ax‖₂² + λ‖x‖₁. Regularised version; soft-thresholding proximal operator. Connects to Bayesian MAP estimation with Laplace prior.
  • OMP (Orthogonal Matching Pursuit): Greedy. At each step, find the column of A most correlated with the residual, add it to the support, project the measurement onto the current support. Requires ~k iterations; O(knm) total.
  • CoSaMP: Model-based iterative algorithm with near-optimal guarantees. Faster than BP in practice.
  • ADMM (Alternating Direction Method of Multipliers): Splits the BP problem into easily solvable subproblems. Primal-dual approach. The workhorse for large-scale CS (CS-MRI uses TV-regularised ADMM). Convergence guaranteed under mild conditions.
The Donoho-Tanner phase transition

For random Gaussian measurements, the probability of exact recovery undergoes a sharp phase transition in the plane (δ = k/n, ρ = m/n). The phase transition curve ρ* (δ) separates the (δ, ρ) plane into: above the curve = exact recovery with overwhelming probability; below = failure. This was derived using the replica method from statistical physics — an exchange of information-theoretic methods between physics and mathematics. The phase transition is sharp: even a small increase in measurements above ρ* goes from guaranteed failure to guaranteed success.

The CS-MRI translation: MRI acquires k-space data (Fourier coefficients of the body image). Natural medical images are sparse in wavelet domain. By randomly undersampling k-space (m ≈ 20% of full Nyquist), compressed sensing reconstruction via TV regularisation + ADMM recovers a diagnostically equivalent image. Scan time: from 10 minutes to 2 minutes. This is now FDA-approved and deployed clinically.

Fields Medal Connections

Several Fields Medalists produced mathematics whose consequences flow directly into compression theory — through sparse recovery, harmonic analysis, variational methods, and optimal transport. Click to expand.

2006
Terence Tao
Compressed sensing — direct contribution

Tao, with Emmanuel Candès and David Donoho, proved the foundational theorems of compressed sensing. The key paper "Robust uncertainty principles: exact signal reconstruction from highly incomplete frequency information" (Candès-Romberg-Tao, 2006) proved that random undersampling of Fourier measurements allows exact recovery of sparse signals via ℓ₁ minimisation. Tao's contribution was the RIP framework and its proof that Gaussian random matrices satisfy it using concentration inequalities from additive combinatorics and his work on arithmetic progressions. His background in harmonic analysis — specifically, the Fourier restriction conjecture — gave him exactly the tools needed to prove that random Fourier measurements are "spread out enough" to satisfy RIP. Compressed sensing is now used in accelerated MRI (FDA-approved since 2017, reducing scan times by 4–8×), single-pixel cameras, radar pulse compression, and astronomical imaging.

1994
Jean Bourgain
Harmonic analysis underpinning sparse recovery

Bourgain's Fields Medal work on the Fourier restriction problem and the geometry of Banach spaces underpins the functional-analytic machinery used in compressed sensing proofs. His restricted invertibility theorem (Bourgain-Tzafriri, 1991) guarantees that any n×n matrix with columns of unit norm has a large subset of columns forming a well-conditioned system — a precursor to RIP. His large deviation inequalities for sums of independent random matrices underpin the probabilistic proofs that random measurement matrices work. The Bourgain-Milman theorem on the geometry of convex bodies connects to the spherical geometry underlying sparse recovery analysis. His contributions are largely invisible in the CS literature but foundational to its proofs.

1994
Pierre-Louis Lions
Variational methods and total variation image processing

Lions's work on viscosity solutions to Hamilton-Jacobi equations and the calculus of variations directly enabled the Rudin-Osher-Fatemi (ROF) model (1992): minimise the total variation ‖∇u‖_L¹ of a reconstructed image u subject to data fidelity ‖u − f‖₂² ≤ σ². This is an ℓ₁ penalty in the gradient domain, which preserves edges (discontinuities in u have finite TV) while penalising texture (smooth oscillations have high TV). The ROF model is the archetype of ℓ₁-regularised inverse problems, and its analysis required Lions's tools from non-smooth convex analysis and the theory of functions of bounded variation (BV space). ROF is used in image denoising, inpainting, super-resolution, and compressed sensing reconstruction for MRI (TV-regularised ADMM). The conceptual bridge from ROF to compressed sensing — that ℓ₁ penalisation promotes sparsity — is one of the central ideas in modern signal processing.

2010
Cédric Villani
Optimal transport in neural and generative compression

Villani's synthesis of optimal transport theory — Wasserstein distances, displacement interpolation, entropy-regularised transport (Sinkhorn algorithm) — has become the mathematical backbone of modern neural compression. The Wasserstein-2 distance W₂(P, Q) = inf_{γ∈Γ(P,Q)} [∫‖x−y‖² dγ(x,y)]^{1/2} measures the geometric cost of transforming the source distribution P into the reconstruction distribution Q. This is a superior perceptual distortion metric to MSE because it accounts for the geometry of the image manifold — moving probability mass to nearby images is cheap, while moving it to very different images is expensive. VAE-based neural codecs minimise a rate-distortion objective that includes Wasserstein-style divergences. Denoising diffusion models for extreme-lossy compression explicitly minimise an optimal transport cost between the source distribution and the reconstruction distribution at a given bitrate.

1998
Timothy Gowers
Additive combinatorics and pseudorandomness

Gowers's Fields Medal work on Szemerédi's theorem via Fourier analysis, and the resulting Gowers uniformity norms, appears in the combinatorial arguments that show random Fourier measurements satisfy compressed sensing recovery guarantees. His methods for proving that certain structured objects behave "pseudorandomly" (i.e., are free of arithmetic progressions) underpin the proofs that random partial Fourier matrices satisfy RIP with high probability. More broadly, his techniques for constructing and analysing pseudorandom structures appear in the information-theoretic capacity arguments for structured measurement matrices — attempts to replace random matrices with deterministic constructions that match their performance.

Abel 2017
Yves Meyer
Wavelet theory — most practical Fields-level contribution

Abel Prize, not Fields — but impossible to omit. Meyer's 1985 construction of the first smooth wavelet basis, and his development of multiresolution analysis jointly with Stéphane Mallat, created the entire theoretical foundation of wavelet-based image and signal compression. Meyer proved that orthonormal wavelet bases with arbitrary smoothness and compact frequency support exist, resolving a conjecture that had been open since Calderón. Ingrid Daubechies then found compactly supported wavelets (1988) — the first practically usable orthonormal wavelet family. Every wavelet used in JPEG2000, digital audio mastering, FBI fingerprint compression, and modern signal denoising descends from Meyer's constructions. His work on sparse representations in overcomplete dictionaries (matching pursuit, basis pursuit) directly connects wavelets to compressed sensing. He is arguably the single most important mathematician for practical data compression.

Section 05

Cutting Edge

The field is in a period of radical change. ANS has displaced Huffman in every new major codec. Learned neural compression matches classical methods at low bitrates and, in the perceptual metric regime, surpasses them.

ANS Revolution (2009–present)

Jarek Duda solved a 40-year problem: achieve arithmetic coding's compression ratio at Huffman's speed. ANS works as a bijection between sequences over a weighted alphabet and the natural numbers. The two variants — rANS (ranged) and tANS (table, marketed as FSE/Finite State Entropy) — have opposite direction conventions: rANS encodes left-to-right and decodes right-to-left (LIFO), while tANS uses lookup tables for O(1) encode/decode. Both are now implemented in SIMD (AVX-512) to saturate memory bandwidth. Used in: zstd (FSE), LZMA2, JPEG XL (VarDCT + Modular), AV1, Apple Lossless Audio Codec (ALAC), Draco 3D geometry. The theoretical analysis is based on the asymptotic analysis of the state distribution and the bijection with natural number representations in non-uniform numeral systems.

Learned / Neural Compression (Ballé et al. 2017–22)

The architecture: encoder network f_θ(x) → y (latent representation) → quantize y → ŷ → decoder network g_φ(ŷ) → x̂. Trained end-to-end minimising R + λD where R is the estimated entropy of ŷ (from a learned prior p(ŷ)) and D is a distortion metric. The quantization y → ŷ = round(y) is non-differentiable; training uses additive uniform noise ỹ = y + u, u ~ U(−½, ½) as a differentiable proxy. The hyperprior model (Ballé, Minnen, Singh, Hwang, Johnston, 2018) adds a second entropy model: a "side information" channel that transmits the scale parameters of the latent distribution, enabling 15–25% rate savings over the factorised prior. Current state-of-the-art: Cheng2020, ELIC, VCT, Entroformer — all use channel-conditional or transformer entropy models. On Kodak benchmark: ELIC achieves PSNR within 0.5 dB of VVC at low bitrates, with dramatically better visual quality (LPIPS) at ≤0.1 bpp.

LLMs as Universal Compressors

Delétang, Ruoss, Duquenne, Catt, Genewein, Mattern, Grau-Moya, Wierstra, Veness, Hutter, Azar (DeepMind, 2023) demonstrated that large language models are powerful general-purpose compressors when their output probabilities are used as the distribution for an arithmetic coder. Chinchilla 70B achieves 0.97 bits/byte on enwik8 — outperforming all algorithmic compressors including LZMA (1.09), CMIX (0.73 on longer contexts), PAQ8 (0.95). On text: ~0.7 bits/byte for English. On DNA: LLMs outperform specialised DNA compressors. On ImageNet: beats PNG. This directly formalises the decades-old conjecture that compression is equivalent to general intelligence. Shannon's 1951 human-subject experiments estimated English at 1.0–1.3 bits/char — LLMs are now below this, suggesting the true entropy of English text is lower than Shannon estimated, or his experimental methodology had systematic upward bias.

Diffusion Models and Perception-Distortion Tradeoff

The Blau-Michaeli perception-distortion tradeoff (2018) formalised a fundamental impossibility: for a fixed rate R, there is a Pareto tradeoff between distortion (pixel-level fidelity, MSE/PSNR) and perceptual quality (distribution-level, FID/LPIPS). You cannot simultaneously minimise both at low bitrates. Classical codecs implicitly optimise MSE — producing blurry reconstructions at low rates. Diffusion-based decoders (Mentzer, Agustsson et al.; HiFiC; StableSR) instead optimise perceptual quality: they use the compressed code as a conditioning signal for a diffusion model that "hallucinates" plausible high-frequency detail. The reconstruction may have lower PSNR but look far more realistic. This is a fundamental shift in what the distortion measure should be — and challenges the axioms of classical rate-distortion theory (which assumes a distortion measure you can minimise pointwise).

Section 06

Open Problems

Fundamental questions remain open at every level of the theory — from the entropy of natural language to the theoretical foundations of neural compression.

Problem 01

The true entropy rate of natural language

Shannon's 1951 prediction experiments with human subjects estimated English entropy at 1.0–1.3 bits/character. Modern LLMs achieve ~0.7 bits/byte, compressing below Shannon's estimate. The formal entropy rate of natural language — as a stationary ergodic process — is unknown. Is English entropy truly below 1 bit/char, or are there systematic flaws in both Shannon's and the LLM measurement methodologies? Related: what is the entropy of protein sequences, source code, and legal text?

Problem 02

Deterministic matrices satisfying RIP

Random matrices satisfy RIP with high probability when m = O(k log(n/k)). But no explicit, polynomial-time constructible matrix is known to satisfy RIP(k, δ) with m = O(k log n). The best deterministic constructions achieve m = O(k²) (DeVore 2007, using algebraic geometry over finite fields). The gap between O(k log n) (probabilistic) and O(k²) (deterministic) has resisted 18 years of effort. A solution would have implications for coding theory, pseudorandom constructions, and the connection between algebraic combinatorics and compressed sensing.

Problem 03

Rate-distortion function for natural images

R(D) is known in closed form for Gaussian sources. For natural images — modelled as distributions on a high-dimensional, curved manifold in pixel space — the true R(D) curve is unknown. Neural codecs empirically trace this curve, but we have no theory. What is R(D) for ImageNet? For medical images? Without knowing the theoretical limit, we cannot say whether current codecs are near-optimal or whether factor-of-2 improvements are available.

Problem 04

Slepian-Wolf with low-complexity codes

The Slepian-Wolf theorem (1973) proves that two correlated sources can be separately encoded at sum rate approaching H(X, Y). The achievability proof uses random codes. Constructing practical codes (LDPC, polar) that achieve the full Slepian-Wolf rate region for arbitrary non-binary sources remains largely open. Practical systems for video compression (Wyner-Ziv coding) operate near the rate region only in restricted Gaussian/Markov settings.

Problem 05

Theoretical foundations of neural compression

Learned image codecs outperform classical methods in practice, but the theoretical understanding is essentially zero. There are no convergence guarantees for the training procedure, no characterisation of the rate-distortion gap, and no understanding of what the encoder network learns in terms of the source statistics. Why does more expressive entropy model architecture (autoregressive > hyperprior > factorised) always improve compression? Is there a theoretical R(D) gap that a perfect learned codec could close, or is the improvement purely empirical?

Problem 06

One-shot and finite-length theory for ergodic sources

Shannon's theorems are asymptotic (block length n → ∞). One-shot information theory (Renner, Wolf, Datta) gives exact finite-length bounds using smooth min/max entropies H^ε_min, H^ε_max. Second-order coding rates (dispersion, √n corrections) are known for i.i.d. and Markov sources, but not for general stationary ergodic sources. The precise √n term in R(n, ε) = H + √(V/n) Q^{−1}(ε) + O(log n/n), where V is the source information variance, is unknown for broad source classes.

Problem 07

Computability gap in Kolmogorov complexity

For natural language and images, the gap between K(x) (uncomputable truth) and C(x) (best compressor) is unknown. Is the gap O(log n), O(n^ε), or O(n)? For specific structured objects (π to n digits, prime factorisation tables), K(x) is provably O(log n). For natural text and images, we have no lower bounds on K(x) at all — meaning we cannot rule out the possibility that a much shorter description exists that no current compressor exploits.

Problem 08

The perception-distortion-rate surface

The Blau-Michaeli tradeoff established the 2D perception-distortion curve at a fixed rate. But the full 3D surface R(D, P) — where R is rate, D is distortion (MSE), and P is perceptual quality (FID) — is unknown. Does this surface have a simple structure? Is it convex? Can it be computed from the source distribution, or does it depend on properties of the human visual system that are not captured by the image distribution alone? This is perhaps the most important open problem for practical lossy compression.
